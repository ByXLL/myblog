DOM的增删改

增

在真实项目中，我们偶尔会在JS中动态添加一些HTML标签，然后把其添加到页面中

documnet.createElement

在JS中动态创建一个HTML标签

appendChild

语法：容器.appendChild(新元素)
把当前创建的新元素添加到容器的末尾位置

insertBefore

语法：容器.insertBefore(新元素,老元素)
在当前容器中，把新创建的元素增加到老元素之前

<!DOCTYPE html>

<html lang="en">

<head>

    <meta charset="UTF-8">

    <title>Document</title>

    <style type="text/css">

        #box1,#box2{ width: 100px; height: 100px; border: 1px solid #ccc;}

    </style>

</head>

<body>

    <div id="box2">box2</div>

    <script>

        // 创建

        var oDiv = document.createElement('div');

        oDiv.id = 'box1';

        oDiv.innerHTML = 'box1';

        // 添加到页面中

        // document.body.appendChild(oDiv);     //添加到body这个容器的末尾位置，div2的后面

        document.body.insertBefore(oDiv,box2);  //新创建的元素增加到老元素之前

    </script>

</body>

</html>

小案例 解析URL地址每一部分的信息

通过动态创建a标签，利用a标签的一些内置属性来获取 

var link = document.createElement('a');

link.href='http://www.baidu.com/?name=admin&password=Aa123456&age=20#root';

console.dir(link);

// hash:"#root" ->哈希值

// hostname:"www.baidu.com" ->域名

// search:"?name=admin&password=Aa123456&age=20"    ->传递的参数

function queryURLParameter(url) {

    var link = document.createElement('a');

    link.href = url;

    var search = link.search,

        obj = {};

    if (search.length === 0) return; 

    search = search.substr(1).split(/&|=/g);     //substr(1) ->从第一个字符开始截取，split(/&|=/g) -> 正则拆分

    for (var i = 0; i < search.length; i+=2) {

        var key = search[i],

            value = search[i+1];

        obj[key] = value;

    };

    link = null;

    return obj;

}

console.log(queryURLParameter('http://www.baidu.com/?name=admin&password=Aa123456&age=20#root'));

删

removeChid

语法：容器.removeChid(元素)
在当前容器中把某一个元素移除掉

replaceChild

语法：容器.replaceChild(新元素,老元素)
在当前容器中，那新元素替换老元素

cloneNode

语法：元素.cloneNode(false/true)
把原有的元素结构克隆一份一模一样的，事件等无法克隆，false：只克隆当前元素本身，true：深度克隆，把当前元素本身以及元素的所有后代都进行克隆，默认为false

<!DOCTYPE html>

<html lang="en">

<head>

    <meta charset="UTF-8">

    <title>Document</title>

    <style type="text/css">

        #box1,#box2{ width: 100px; height: 100px; border: 1px solid #ccc;}

    </style>

</head>

<body>

    <div id="box1">box1</div>

    <div id="box2">box2</div>

    <script>

        var box1 = document.getElementById('box1'),

            box2 = document.getElementById('box2');

        //document.body.removeChild(box1);     //移除box1这个元素

        //document.body.replaceChild(box2,box1);    //box2替换掉box1

        var box3 = box2.cloneNode(),

            box4 = box2.cloneNode(true);

        document.body.appendChild(box3);    //只克隆元素本身

        document.body.appendChild(box4);    //后代元素一起克隆

        box2.onclock = function () {

            alert('box2点击事件');                  //克隆后的元素不带有原来元素的事件

        }

    </script>

</body>

</html>

改

[set/get/remove]Attribute

给当前元素设置/获取/移除元素的自定义属性（一般操作的是自定义的属性）
box.setAttribute(‘myIndex’,0)
box.getAttribute(‘myIndex’)
box.removeAttribute(‘myIndex’)

使用xxx.index=0和xxx.setAttribute(‘index’,0)的区别？

xxx.index：把当前操作的元素的当做一个普通对象，为其设置一个属性名

在对象的属性中添加一个index的自定义属性，而不影响DOM

xxx.setAttribute()：把元素当做特殊的元素对象来处理，设置的自定义属性是和页面结构中的DOM元素映射在一起，相当于添加内置属性

JS中获取元素对象，我们可以把它理解为两种角色

  • 与页面HTML结构无关的普通对象
  • 与页面HTMl结构存在映射关系的元素对象

元素对象中的内置属性，大部分都和页面的标签存在映射关系：
xxx.style.backgroundColor = ‘xxx’此时不仅把JS中的对象对应的属性值改变了，而且也会映射在页面的HTML标签上（标签中有一个style行内样式，元素的样式改变了）

xxx.class = ‘xxx’此时不仅是把JS中的属性值改了，而且页面中的标签增加了class样式类

元素对象中的自定义属性：xxx.index = 0
仅仅是把JS对象中增加了一个属性名（自定义的），和页面中HTML没有改变

xxx.setAttribute()通过这种方法设置的自定义属性和之前提到的内置属性差不多，和HTML结构 存在映射关系（设置的自定义属性可以呈现在结构上）

typeof

使用typeof检测，返回的结果是一个字符串，字符串中包含的内容证明了值是属于什么类型的
[局限性]
1、typeof null 不是”null”而是”object”，因为null虽然是单独的一个数据类型，但是它原本意思是空对象指针，浏览器使用typeof检测的时候会把它按照对象去检测。
2、使用typeof无法细分出是数组还是正则，因为返回的结果都是”object”。

typeof {name:"xiaoming"} => "object"

typeof function (){} => "function"



/*简单实例*/

Number()

把其他类型值转换为number类型的值
使用Number()换的时候，只要字符串中出现任何一个非有效数字字符，最后结果都是NaN。
在把引用数据类型转换为number时，会隐式地把引用数据类型转换为字符串（.toString()），再把字符串转换为number。
通俗地记忆，除true、false、null外，只有有效数字和只有一项且为有效数字的数组，才能返回数字，其余全部返回NaN

Number(true) => 1

Number(false) => 0

Number(null) => 0

Number(undefined) => NaN



Number("12.3") => 12.3

Number("12.3.4") => NaN     //注意第二个点，使得数值变得没有意义，就不是一个数字

Number("1px") => NaN



Number([]) => [] -> ""  "" -> 0

Number([1]) => [1] -> "1" "1" -> 1

Number([1,2]) => [1,2] -> "1,2" "1,2" -> NaN



Number({}) => NaN       //({}).toString() => "[object Object]"

Number({name:'xiaoming'}) => NaN

parseInt()

数据类型值转换为number，和Number()方法在处理字符串的时候有区别
提取规则：从左到右依次查找有效数字字符，直到遇见非有效数字字符为止（不管后面是否还有），把找到的转换为数字。

Number("1px") => NaN

parseInt("1px") => 1

parseInt("px1") => NaN

parseInt("1px2") => 1

parseInt("1.2px") => 1

parseFloat()

在parseInt的基础上识别小数

parseInt("1.2px") => 1.2

NaN

Not a Number：意思是（不是一个数），但是属于number类型。
NaN == NaN ：false，NaN和任何值都不相等。

isNaN()

返回结果：true或者false。
用来检测当前这个值是否为非有效数字（是不是不是一个数），如果不是有效数字检测结果是true，反之，是一个有效数字则为false。

isNaN(0) => false

isNaN(NaN) => true

isNaN("12") => false -> isNaN(12) => false

isNaN(true) => false

isNaN([]) => false

当我们使用isNaN去检测一个值的时候，检测的值不是number类型的，浏览器会先将值转换为number类型，然后再去检测。

一个!为取反，!!为取两次反，相当于没有取反，但是却把其他数据类型转换为布尔值，与Boolean ()相同的效果

null与undefined

null：空，没有
undefined：未定义，没有

空字符串与null的区别

打个比方：种树
空字符串：相当于挖了个坑，却没有种
null：坑都没有挖
空字符串相对于null来说，空字符串在内存开辟了空间，而没有存值。

null与undefined的区别

打个比方：存钱买房
null：暂时没有钱，但是产生了一个构想，慢慢工作，以后会有钱了买房，结果可能实现或不实现。
undefined：白日梦，什么都不会发生，你已经失业了，赚不了钱了。

object

对象的属性名是唯一性
每一个对象都是由零到多组属性名（key键）：属性值（value值）组成，每一组键值对中间用逗号分隔。
属性：描述这个对象特征的描述，例如（你的名字），字符串类型。
属性值：这个对象特征的描述的具体值，例如（小明），任意类型。
对象的属性名可以是字符串或者数字格式。
对象的属性的属性值可以是任意数据类型。
如果操作的属性名在对象中不存在，则获取的结果为undefined。

  • 操作属性的办法
    • 对象名.属性名：忽略属性名的单双引号。
    • 对象名[’属性名‘]：不能忽略单双引号。
  • 设置修改属性的方法
    • oStu.sex = '男'; ->未定义过的属性，为设置对象的属性
    • oStu.age = '21'; ->已经定义过的属性，为修改对象的属性
  • 删除属性
    • oStu.age = null; ->假删除，让其属性值为null，但是属性还在对象中
    • delete oStu.age; ->真删除，把整个属性在对象中移除
